## Taxonomie de référence

### Types d'exigences
- F  : Fonctionnel (ce que le système doit faire)
- T  : Technique (comment le système doit être construit)
- O  : Organisationnel (gouvernance, MCO, SLA, équipe)
- C  : Contractuel (délais, pénalités, réversibilité)
- R  : Réglementaire (RGPD, HDS, normes, certifications)

### Domaines BDAT
- B  : Business (processus métier, SLA, continuité)
- D  : Data (données, flux, souveraineté, MDM)
- A  : Application (ERP, CRM, applicatifs spécifiques)
- T  : Technology (infrastructure, réseau, sécurité, cloud)

### Priorités
- OBL : Obligatoire (shall / doit / must / obligatoire / éliminatoire)
- SOH : Souhaitable (should / devrait / recommandé)
- OPT : Optionnel (pourrait / nice-to-have)
